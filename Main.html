Process For How I Create Games

Created 2024/09/21

By: Clyde Pestano

Creating a games is a very difficult process, a lot goes into not only planning but managing where things are, the art, sound design,

and writing the actual code. I have been creating games for while now and my own process to how I make games I would like to share

Here is a list of that process

  1. Start of with an idea
  2. Find resources related
  3. Figure out how it's going to look(3D/2D art direction)
  4. Start scripting and organizing
  5. Audio Design

Start of with an idea

What do you want your game to be about? rpg, roguelike, shooter, bullet-hell,even better would be a combination of these things

But you first goal should trying to created something you think would be fun, look to other good games as inspiration

Find resources related

Finding the resources to learn how to make your games is very important especially if you have never made the game before.

This is also applies if you don't have much knowledge on the code your working with, for that look at

Resource Links

Figure out how it's going to look(3D/2D art direction)

The look of a game I would say is the most important part of the game. The look is able to draw in players before they even see the game.

If you're doing 3D look into how to make a character and proportions of them. Make the designs unique and pleasing to look at

if you doing pixel art look into sprite sheets and color palettes to make your game look good in the style you want

Start scripting and organizing

You can now start scripting the mechanics of the game. always make folders for important game mechanics like movement,

enemy behaviors, animation handlers and more, for resources on how to learn to use the engine there one is resource link

that can be used for unity

Audio Design

Audio is one of the most underrated features in video games good audio can enhance how a game feels by a lot but bad audio design

can suck the life out of the game and make it feel dull. when doing audio design focus on the context of what the audio is for

for example if you hit metal i should sound like you hit a pan or anything metallic

Conclusion

This is the way I go about making games; however you are never limited to only following one process. maybe you want to finish the visuals

for a game first then implement the mechanics that is perfectly fine, as long as you are following a good plan you should be able to succeed
